Cinestron Geting Started¶

Upon starting you would of recieved a username and password. We use a unified login system at Cinestron. No need to remember multiple passwords and accounts. Server, Im, email etc are all handled together.

  • Username

    All usernames are first name initial then last name an example for Bill blogs would be: b.blogs b.blogs@cinestron.com password : ***

The username serves as access to Cinestron network and also your email.

  • Yandex Email

    Setup your email account on yandex.

    https://mail.yandex.com/

    Use your cinestron email and password

  • Discord IM

    We use a private discord server for Instant messages and video calls. Use the same email and password to create an account if one is not allready setup for you.

    ‘https://discordapp.com/invite/Pt7wsQg’ ‘https://discordapp.com/login’

Network info¶

Active Directory (AD) Key Points…¶

  1. the AD contains user accounts, user groups, computer groups and group policy objects (GPOs)
  2. group policy objects are windows settings which can be applied to different groups of users and computers
  3. AD servers are split across what is known as “sites” (locations like Russia, UK etc.)
  4. there are 2 types of sites: primary domain controller (PDC) site and secondary domain controller (SDC) site
  5. a domain is what contains the AD and links all the computers together
  6. there can only be one PDC, whereas the SDCs can be numerous
  7. the AD automatically replicates across the network (between PDC and SDCs) every 15 mins

Computer Naming Conventions…¶

  • the naming convention is composed of 4 parts: the first part is a 3-letter (except for personal computers which use 2 letters) functional purpose of the machine, the second part is a 4-letter type of the machine (server, desktop, laptop etc.), the third part is a 2-letter country code and the fourth part is a 2-digit index

  • the first part variations are:

    PDC = primary domain controller

    SDC = secondary domain controller

    CLU = cluster machine (hosts a collection of various virtual machines)

    CPU = CPU-only machine (virtual in most cases)

    GPU = GPU and CPU machine (physical) first letters of name and surname (eg: AT) = individual user computer

  • the second part variations are:

    HOST = physical machine acting exclusively as a host for virtual machine(s)

    SERV = virtual/physical server machine

    TERM = terminal virtual/physical machine serving exclusively for the purpose of remote access

    WORK = physical workstation

    DESK = physical desktop

    NOTE = physical notebook

  • the only exception to the naming convention rule is the primary domain controller because there can only be one in a network

  • all names are in capital letters and separated by dashes

  • the maximum length of a computer name in windows is 15 characters

IP Address Considerations…¶

  1. composed of 4 numbers, each ranging from 0 to 255, eg: 101.212.78.2
  2. all IP addresses on the Cinestron network start with 192.168.
  3. the third number delimits the type of group of device/computer: 1 = routers, 2 = main servers, 3 = manually assigned (Russia), 4 = manually assigned (UK), 101 = automatically assigned (via DHCP) in Russia, 102 = automatically assigned (via DHCP) in the UK
  4. the fourth number is an index of the device/computer in the group
  5. in order to separate the index numbers for the first 2 group types between Russia and the UK and prevent them from colliding with each-other the indexing convention is split into groups of 10: Russia = 1 to 10, UK = 11 to 20
  6. automatically assigned IP addresses are leased (given out by the DHCP server) for 7 days, after which they are reassigned (the same happens with external internet IP addresses given out by the ISP, hence the need for DynamicDNS services)
Examples:

192.168.1.1 = main router in Russia 192.168.1.11 = main router in the UK

192.168.2.1 = main server in Russia 192.168.2.11 = main server in the UK

192.168.3.1 = workstation in Russia 192.168.4.1 = workstation in the UK

192.168.101.123 = random automatically assigned IP in Russia 192.168.102.213 = random automatically assigned IP in the UK

Port (Forwarding) Considerations…¶

  1. network ports in TCP and UDP range from number 0 to 65535
  2. if a custom port needs to be forwarded to access a service or application from the internet a convention similar to the one for IP addresses exists
  3. the port is composed of 5 numbers: first number = 1-digit type of service/app, second and third number = 2-digit mapping of the IP group, fourth and fith number = 2-digit mapping of the IP index
  4. currently the accessible custom service/app port types are: 1 = RDP, 2 = RAdmin
  5. ports cannot start with zero, therefore for the first 9 types of services/apps the port numbers are composed of only 4 digits
  6. for instance, in order to access RAdmin installed on a computer with the IP address 192.168.3.17 (a machine with manually assigned IP in the UK), the port number would be: 2317
Examples:

10201 = RDP access to main server in Russia

10211 = RDP access to main server in the UK

20301 = RAdmin access to workstation in Russia

20311 = RAdmin access to workstation in the UK

  1. because the maximum port number is 65535, there can only be up to 6 different custom service/app types, up to 99 device/computer groups and up to 99 device/computer indexes
  2. several ports are reserved for default system services/apps, but they do not collide with the Cinestron port numbering convention
  3. most services/apps are handled by default system ports, therefore 6 custom types are enough

Currently the reserved default system ports are:¶

  • 21 = FTP
  • 80 = HTTP
  • 443 = HTTPS
  • 1723 = VPN
  • 8082 = Deadline Web Service
  • 8083 = Deadline Remote Connection Server (RCS)
  • 27017 = MongoDB
  • 45124 = RV

    X(user.name)

    Password:userpassword

    Root of network in file browser: \192.168.2.1network

Network folders structure also maps to the Z: drive on your local machine.

  • VPN filepath example

    \192.168.2.1networkprojectsinternal201709_september18_vivo \z:networkprojectsinternal201709_september18_vivo

  • FTP

    cinestron.dyndns.org X(user.name) password:userpassword

Web Server direct downloads¶

Sometimes large files are best downloaded through the browser.

Especially if they don’t exist in current project. Eg. image library’s.

cinestron.dyndns.org/files/render.zip

Troubleshooting server connection¶

*Pinging the server from command prompt
“ping cinestron.dyndns.org”
*To refresh server speed in command prompt
“ipconfig /release”

Then restart. If these steps fails contact a.turusov or j.gaffney.
